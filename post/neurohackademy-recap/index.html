Neurohackademy - Intro to the Languages We Used

This year I was lucky enough to be one of the attendees of Neurohackademy, a two-week workshop focused on reproducible and shareable neuroscience. The first week of this workshop featured a dazzling array of speakers from a variety of research backgrounds, including psychology, computer science, and biomedical engineering, who led demonstrations on data science tools and methods. The second week was spent putting these tools to use in groups, hacking on different projects that were then shared with everyone on the last day of the workshop. Overall it was an experience that was both intellectually and personally fulfilling, and I strongly recommend the program to other students interested in open and accessible neuroscience.

When I got back there was some curiosity about the program. Besides questions about the experience, the number one thing that was asked was “Do you need to be a programmer to attend?”. While the answer is “no,” I did find myself struggling with imposter syndrome when trying to use a tool for the first time that most of everyone else seemed to be fluent with (üêç). Luckily, everyone at the workshop- both instructors and attendees- were patient, kind, and willing to sit down to walk through the very basics of how to get started. Unfortunately, we all can’t be at Neurohackademy all of the time, and if you’re like me, you might find the very first steps of using a new tool to be the hardest- and the documentation to be the most obtuse. So I wanted to write a post that works through the first baby steps to getting started with the tools that we used during the workshop. In this post I will introduce the different programming languages we used (bash, R, Python).

If you are interested in the overall experience of Neuorhackademy, I recommend reading the thoughts shared by several of the attendees and instructors: Amanda Easson, Kristian Eschenburg, Ariel Rokem (organizer/instructor), Tal Yarkoni (organizer/instructor).

Jump to First Steps Tutorials

A Disclaimer

I am neither a computer scientist nor a programmer. These tutorials are not meant as comprehensive lessons, but rather quick-start guides that will help you get up and running toward reaching your own coding goals. If you are familiar with these languages and think that I missed something important, send me an email and I will add your comments here with credit.

Advice About Coding

It can be intimidating to see other people who are ‘good’ at coding. At Neurohackademy it was hard to resist dissolving into a quivering ball of self-consciousness watching my friends program a brain parcellation fragmenter that spits out beautiful rainbow brains in python while I struggled to successfully print “Hello world” to the console. But keep in mind that the term ‘programming language’ is extremely apt- just as you wouldn’t be expected to be fluent in German after a vacation in Berlin, your first foray into R or C++ will be a bit of an uphill climb as you try to learn complex declension rules and battle with errors over when to use ‘du’ or ‘Sie’. And while taking a class can help you learn the basic grammatical rules quicker, there is nothing like fully immersing yourself in an environment to become fully fluent in a language. There is a beautiful truth to coding- you can definitely do it, with practice and patience. But if you really want to make things stick, nothing beats working on a project of your own.

"There is a beautiful truth to coding- you can definitely do it, with practice & patience."

That being said, here is my advice for those getting started with coding, or learning a new programming language:

  • Learn a language because it has utility for you.
    Coding is a time sink. And just like spoken language, it is easy to lose the fundamentals if you don’t practice using it often. Invest your time wisely into learning a programming language that you will use for your own research and/or that is used commonly in your desired career.
  • Use the internet.
    The communities for popular languages are massive, and massively online. Stack Overflow, Cross Validated, and the Python forums are just three popular examples of online resources to help you get started or get unstuck. Whenever you have a task or error code you can’t figure out, fire up your favorite search engine and see if the internet has already solved your problem.
  • Remember- even the pros’ code breaks.
    One of my favorite things about Neurohackademy was how honest and nonchalant the instructors were when their code broke, or when they couldn’t figure something out. It was refreshing! Broken code doesn’t mean you’re not a ‘real’ programmer, and asking for help isn’t something to be embarassed of. Shit happens and shit breaks, no matter how long you’ve been coding.
  • Check your paths.
    The #1 source of broken code in every language that I’ve used is a path not being specified correctly (unfamiliar with this term? Jump down to the Bash tutorial). Do yourself a favor and form a habit of double-checking paths as the last thing you do before you run your code, and the first thing you do when debugging.

I hope these points help as you work through the first steps tutorials below, and throughout your adventures in coding. The ideal is coding as a fun, creative experience rather than an exercise in frustration.

With that unsolicited advice out of the way, let’s move on to learning the basic first steps of working with the different programming languages that we used at Neurohackweek. I’d recommend that anyone interested in a career in psychology, neuroscience, or data science gain experience with these three commonly used languages.

Bash

Let’s get this out of the way first- whether bash is technically a programming language is a debate similar to if tomatos are fruits and if Pluto is a planet; i.e., technical arguments for specialists in those fields who are upset over the layperson usage. As a psychologist and not a computer scientist, I’m going to refer to it as a programming language for simplicity’s sake as you write in its idiosyncratic language to tell your computer to do something.

That being said, what is bash? Bash stands for Bourne-again shell, and while the first two letters represent a funny inside joke, the latter two- standing for ‘shell’- tell you exactly what it is. Computers that run on Unix operating systems can open up shells, or command-line interpreter interfaces, to directly access the guts of the operating system. All bash is is a specific type of Unix shell that runs on your system and allows you to type commands that tells the system directly what you want it to do. You can tell you computer what to do using a shell either by typing directly on the interface’s command line, or by writing and running a shell script- a bunch of code commands that you save as a .sh (shell) file.

Apple computers come out-of-the-box with an application called Terminal, which is a bash shell. I think Windows 10 released a bash shell for use as well, but being as how I haven’t touched a Windows machine since 2010, this first steps tutorial will be MacOS-oriented.

Finally, as a psychologist I typically use bash for three reasons: creating and navigating file structures; writing and running scripts that make my life easier; and accessing command-line programs, like python or docker. The first section of this bash tutorial will help you be able to perform the first activity and the following section will help get you ready for the second, while the third activity is addressed in the first steps tutorial on python.

Step One: Opening a bash shell & navigating around

To get coding in bash, the first baby step is of course to open a bash shell. On a Mac, how would you open a bash shell? Hover over the spoiler for the answer. Open the ‘Terminal’ application.

Once you have successfully opened a bash shell, it will look something like this:

This is your window into the guts of your computer’s operating system, much like those portholes that scientists put into the sides of cows to study their digestion. The area where you see a blinking cursor- probably after YourUserName$- is your command line. Here is where you will type the things you want your computer to do, and bash will interpret these commands and submit them to the operating system.

Now it is time to learn one of the most important lessons of coding- how to navigate a file system. Even if you have never used a shell before, you already know how to navigate a file system using the graphical interface of the MacOS! The Finder application is just a file navigation system that allows you to interact with your computer’s files by clicking and dragging pretty pictures of folders, rather then typing directions on a command line. So when we are in the bash shell, try to keep in mind that all we are doing is navigating through Finder using the keyboard rather than the mouse.

Let’s start with a basic, important command. Type pwd on your command line in your bash shell. This stands for print working directory, and tells you the full path name of the folder where you currently are submitting commands from in your computer’s file directory. When I type this command in my terminal, I see /Users/rx. Since I’m not using a graphical interface like Finder, however, this doesn’t help me see what files currently exist in the /Users/rx folder. To do so, type ls, which stands for list.

Here are a list of the folders and files in my system that “live” in the /Users/rx folder. You might have noticed that some of the words are printed in different colors; I’ve set my custom colors so that folders are printed in teal and files are printed in white. This distinction between the two is important because we can change our working directory into different folders.

Let’s try changing directory to a familiar place; the desktop. Type cd Des, then press the tab key. Did the command line auto-complete the word ‘Desktop’ for you? This is a neat trick that will save you a lot of typo-induced heartache; bash can complete the names of folders, files, and paths that it recongizes in the file system. Now try listing the files in this folder.

A brief aside

What happens if there are two folders in my current working directory that begin with the same letters? If I press tab once, nothing will happen (except an upset computer noise). But if I press tab twice, the following is printed:

Bash helpfully prints out all of the matching options so that I can narrow down my command to the intended folder path. The double tab can be used to list the contents of any folder path from the command line in case you forget where you are in your directory structure.

Certain characters have special navigational meanings in bash. . means here, and .. means the folder containing the one that you are presently in. So, for example, if I wanted to change directory back from my Desktop into my /Users/rx/ folder I could type cd .. instead of the much longer cd /Users/rx/. But maybe I just want to check what files are in the /Users/rx/ folder without actually changing my present directory there. What would I type to achieve this? ls .. .

Finally, specific bash commands can be augmented through the use of flags. Flags follow the command being called, or written on the command line, and come in the format of -letter, where the letter stands for the particular command augmentation. Let’s clear this up. Before when I wanted to list all of the files in my /Users/rx folder, I called the ls command while that was my working directory. It printed out a list of all the visible files in that folder to the console, as expected. But I can make ls display other things depending on the flags that I specify. For example, the -l flag says that I want the ouptut in long format, which means that additional information, like file access permissions, will be printed. So, the command for this now looks like ls -l.

If I was sorting through all of my manuscript_definitely_final.doc files manually and I completely forgot which one was actually the most recent, and therefore most correct, version, I could use the -t flag, which sorts the printed files by modification time. But unfortunately when I do that, the files are listed without any information about when I actually had edited them. Thankfully, we can combine flags on the command line, and so by passing, or submitting, the command ls -tl, I get the following output that both lists the files by modification date and time and tells me what that date and time was:

These basic commands are the building blocks of using bash, and are necessary to learn in order to use a language like python or a service like git. I recommend taking the Code Academy course on learning the command line to further your understanding of bash and command line functions.

Step Two: Installing and using a text editor

When I first learned about bash it wasn’t brought up how important picking a text editor is to maintaining your sanity while coding. Whether you want to read, write, or run scripts, in bash or python, on your local machine or a high performance computing cluster, you will want to pick the best tool to help you. For a while I stubbornly only used nano, a text editor that comes prepackaged within the Terminal application (I feel like I must emphasize again that I am not a computer scientist). Nano can get simple jobs done just fine, without needing to load and launch an exernal program. Let’s walk through what it does.

I have a list of subjects in a file called subject_list.txt. To read this text file inside nano, simply type nano subject_list.txt. This is what it looks like inside the text editor:

Very no-frills. The text is all the same color and the interface is very simple, but it is good for managing basic tasks, like adding in another participant to the list by hand or fixing a typo quickly in a script. Once you are finished working with the text file press (on a Mac) control + x to close nano, and it will ask you if you want to save changes and/or save the file under a different name.

Nano is great for simple, quick tasks, but if you are writing a script something with more functionality is better to use. I use Atom. To use a text editor that is not already available in your bash shell, you first need to download the editor, then make it available for bash to actually run. So, the first step is to follow the instructions on the Atom website to download the correct version for your operating system.

When the download wizard prompts you to click+drag the Atom application into your applications folder in a separate pop-up window, that process does not work correctly. Once you have downloaded Atom, launch the Application. From there, on the top navigation bar, under 'Atom', select 'install shell commands'. This will allow you to call Atom from inside your bash shell.

Now from the command line I can pass `atom subject_list.tx` to open that text file in Atom. But again, I will primarily use nano for little, quick tasks, and Atom for scripting. Below I have posted a script that I wrote to segment brain areas and create masks from those segmentations, both in nano and Atom.

The difference is visible- Atom color-codes different parts of a script to ease reading and add another level of double-checking for code. For example, if I am trying to establish a new variable and do not see that that part of my script is printed in red, I know that I have typo'ed something somewhere. But it isn't just the color-coding that makes using an external text editing program preferential- there's additional functionality, like error checking, the ability to select and edit multiple lines simultaneously, and the possibility to add in even more functionality through their suite of packages. There are other text editing programs out there (another popular option is Sublime Text), so you should find and use whichever works bests for you and your coding style.

Next Steps: Learning more bash commands & Editing your bash profile

The Code Academy course on learning the command line is a great resource for furthering your understanding of bash and its basic commands.

You can personalize your bash experience, like I have done with my modified cursor icon and color scheme, by editing your bash profile. This blog post reviews the basics of what a bash profile is, and provides an example bash profile with a variety of different modifications that you can copy for your own use (including editing basic bash commands and setting key shortcuts). Certain text editors may require you to edit your bash profile to be able to use that editor from bash. It can be fun to play around with and customize, but the important thing is to know that the bash profile exists- especially since it is a hidden file, which means that it won’t show up with the ls command (but will with ls -a!).

TL;DR

Bash commands are the basic building blocks for many processes in data science. Being able to comfortably open a bash shell, navigate a file system, and specify a path are good first steps to understanding this language. If you plan on using the command line and writing scripts regularly in your work, you should download a text editing program. Atom and Sublime Text are two popular options. You can further your experience with bash by customizing your bash profile and taking the Code Academy course on learning the command line.

R

According to the R website, “R is a language and environment for statistical computing and graphics”. It has increased in popularity within the field of psychology in recent years as a free, open-source, fully modular alternative to the costly and obtuse SPSS statistical programming package primarily used in the field for many years. One of the major benefits to R is the vast online community that works with it, develops packages for it, and writes documentation and tips. Other benefits include a multitude of tools for (very fancy) data visualization and a variety of packages developed to reduce the learning curve to using this language. I recommend learning R for those who primarily work with behavioral data, although recently there have been R packages written for neuroimaging analysis, such as Neuropointillist. Speaking of Neuropointillist, fellow Neurohackers wrote companion code for the package as well as detailed tutorials on its use- check it out if you want to work with neuroimaging data in R!

Step One: Downloading R and R Studio

Go to the Comprehensive R Archive Network (CRAN) to download R from a mirror hosted close to your location. Once R is successfully downloaded and installed, the next step is to install the free, open-source version of R Studio. R Studio is an integrated development environment (IDE) for R- basically, a software program that allows you to edit and debug source code in one nice-looking visual package. While you can of course choose to work in the base R program, R Studio adds visual appeal and functionality that make coding and debugging in it much easier.

Once you download and install R Studio, you can launch it to see what the graphical user interface (GUI) looks like. You will see four windows:

  • The Console is the command line. Type getwd() in the console and pass the command to print the current working directory (analagous to pwd in bash).
  • The Script Editor is where scripts can be loaded, written, and saved. There are a variety of different types of scripts that can be written in R Studio, including Shiny applications (interactive web applications), R markdown files (which can spit out fully reproducible .pdf or .html files), and blogdown projects (which I am using right now to create this post- meta!).
  • The Environment Pane has tabs for the global environment and history of commands passed in your R workspace. The enviroment tab lists all of the variables in your workspace. Type a <- 3+2 in the console. You will see the variable a and its value appear under in the environment tab. The history tab is a log of all the commands that you have entered in a working session; this should display both a <- 3+2 and getwd() if you practiced passing those commands to the console.
  • The Display Pane contains miscellaneous tabs displaying information about your workspace, as well as a viewer where drawn objects (like charts or figures) appear. The files tab is the graphical depiction of where you currently are in a file structure. The packages tab lists all of the R add-on packages that you have installed. And the help tab will populate with documentation from CRAN.

Step Two: Navigating a file system & Function arguments

Now you should try to navigate around in a file system, just like we practiced in bash. The commands in R are different, and also take on a slightly different format. Specifically, while R commands also are in the format of calling a function on an object, whenever there is a command that you want to execute, you will first type the name of the command followed by the object in parentheses (whereas in bash the object was just typed immediately after the command). We can try this by navigating to the folder above our Desktop folder, like we did in bash. setwd is the R command for changing our current working directory. What is the rest of the command? setwd(‘/Users/rx/’) . Now, use the getwd() command to print your current path. I see [1] "/Users/rx" printed, which means that I successfully set my working directory to the correct folder.

Getting errors?

You may be running into an error if the command is not properly formatted- something like Error: unexpected ')' in "setwd(Desktop/)". This is because the path name is considered a string, or an arrangement of alphanumeric characters meant to be interpreted by the computer as a sole unit. To fix this error, use single '' or double "" quotes around the path name to indicate that it is a string. Note that this also works using the special characters that we learned in bash for ‘here’ . and ‘the folder containing this one’ ... So, if from the Desktop/ folder you type the command setwd('..'), you will change directory into the folder containing the Desktop.

You can also modify the execution of commands in R, similar to how using flags modifies bash commands. These modifications will get placed inside of the parentheses with the object. To practice this, change directory back to your Desktop folder. As we did before in bash, we will list the contents of this folder using a function called list.files(). You will see the files located in your Desktop printed to the console, as expected. But we can modify this command to spit out additional information, and one of the benefits of using R Studio is that we can read all of the different possible arguments that could modify this function within R Studio itself. Type ?list.files; the ? character before the function name tells R Studio to display the CRAN documentation for that function in the Help tab.

The Usage section displays how the function and arguments should be formatted, as well as the default options that are run when you do not add any arguments to the function. Here, you can see why we can type getwd() with empty parathenses and produce the correct output- because the default path name input in the function is path = ".", or ‘here’.

The Arguments section describes the different arguments that you can include in the command. Here we can see that the argument full.names, which is by default set to FALSE, is an argument that determines whether or not the full path of a file is displayed instead of the file’s name. Try setting that argument to TRUE and running the command (command is under the spoiler) list.files(full.names=TRUE). You should see output that looks different than when you pass that command without including that argument. As a final practice, try to list the files in a different directory on your computer without changing your current working directory to that location. Remember, follow the format for arguments specified in the Usage section and put quotes around strings. My command for listing the files in my Documents folder is under the spoiler: list.files(path=‘/Users/rx/Documents’)

Step Three: Installing and loading packages

As mentioned before, the real power of R is all of the various packages, or compiled collections of code and functions, that others have developed. Installing packages is fast and easy and done via the command line. Let’s practice using a package of interactive introductory lessons for R, called Swirl.

The first thing that we need to do is download the package from CRAN. Type install.packages("swirl"). R will then fetch and install the Swirl package.

We aren’t done quite yet. While we have installed the files necessary to use Swirl, we still haven’t loaded it in our working environment- this means that we still can’t pass Swirl commands from the console.

A brief aside

Try passing ?swirl to the console. R should have printed “No documentation for ‚Äòswirl‚Äô in specified packages and libraries: you could try ‚Äò??swirl‚Äô”. Because we have not loaded Swirl yet, R cannot search its commands to display the help information. But another benefit of R is that we can still search any command from any package that we have installed by typing ?? before the command name. If you type ??swirl the help page will refresh and show you a list of possible matching commands across all of your installed packages. This is a very useful function because as a regular R user you will probably install a very large amount of packages, and by searching in this way you can pinpoint exactly which package you would like to load to your current working environment.

To actually use the Swirl package, we have to load it. Type library(swirl) to get started with this package. Swirl will print a welcome message once it is successfully loaded, and you should see the Swirl package underneath your Packages tab on the Display Pane. If you see this message, congratulations! You have started on the highly addictive path of installing R packages and will soon have a veritable dragon’s hoard of packages in your library to play with. You only need to install a package once, but package libraries need to be loaded for every R session (or R script) that you would like to use them in.

If you will be using R for data science, here are a handful of popular packages to get you started (and remember, you can install them all using the install.packages() command right in R studio):

  • tidyverse, for manipulating data sets with ease
  • ggplot2, for beautiful data visualization and enhanced customization of display options
  • purr, for enhaced functional programming
  • psych, for a range of analyses commonly used in psychology

Next Steps: Learn to code in R

If you have downloaded the Swirl package and loaded its library to your working environment, you can call swirl() to take a series of interactive lessons on popular basic topics in R. Everything from learning basic commands, to running regressions, to producing and customizing plots. I highly recommend Swirl to learn how to code in R within the natural environment of R Studio. Before you get started you can read about the different Swirl course offerings to find one that interests you, then install that course module in the swirl() menu options when you are ready.

Another great resource is the free online textbook, R for Data Science. This text is informative for both novice and advanced users of R looking to work with complex data types. As one of the co-authors is the author of the tidyverse series of packages, this text also discusses how to use tidy commands to manipulate data in a more friendly way than base R permits.

TL;DR

R is a free, open-source, widely-used language and a great option to use for data science. Some of the many benefits of R include the ease of installing packages hosted on the Comprehensive R Archive Network (CRAN), the variety of packages available for download, and the comprehensive, well-written package documentation. Downloading the interactive development environment (IDE) R Studio is strongly recommended. Recommended resources for learning coding in R are the tutorials in the Swirl package and the R for Data Science online textbook.

Python

Ah, Python. The language preferred by Serious Computer People across a diverse rainbow of fields. The language that my ignorance of cost me an internship that I really, really wanted. The language that I wish I had been taught during undergrad rather than SPSS or about mountains of unreplicable social psychology effects. I say with a wistful sigh that if I could go back and change my educational trajectory, I would have started with learning Python. With that being said, let’s get you started on your on education in this prevalent, versatile language.

Step One: Installing Anaconda & Launching Python

It is possible that you may already have python installed on your machine. Open a bash shell and type which python. If python is installed somewhere on your machine, the path where it is installed will be printed. If python is installed, type python --version to check which version you have. You could then launch python from the command line by typing python, and if you see the >>> appear before your cursor, you have successfully launched python and can begin coding in it.

But let’s do something slightly more complicated now to make things a lot less complicated later on. That complicated thing is installing Anaconda, a data science platform and package management system. Follow the instructions on the Anaconda site to download the program for your particular OS (and you can check out the Anaconda documentation for more info). WARNING: Download either the version for Python 3.6 OR the version for Python 2.7, but NOT BOTH. I recommend that you download the 3.6 version, as it is more up-to-date (and Anaconda will download a version of Python to match your choice). Unless you know better, you can accept the defaults in the installation wizard and proceed with the install, then launch the Anaconda Navigator and click on the ‘Environments’ tab.

The above is similar to what you will see under your ‘Environments’ tab, the only difference being that I have already created new environments to work in. This tab is one of the reasons why Anaconda makes your life less complicated later on; as you continue to write and run code, you may often run into version issues- your labmate’s analysis script was written in Python 2.7 but your favorite new data vis package requires 3.6 to run, for example. The Anaconda Navigator allows us to create self-contained environments with different versions of Python to work with by clicking on the ‘Create’ button at the bottom.

With different environments created, accessing different versions of Python is as simple as launching a new Terminal window from within Anaconda Navigator.

I currently have one environment creatively named ‘oldpy’, where Python 2.7 is installed. Here, I am opening a Terminal in that environment and one in my current user (rx) directory to show the difference in the Python version installed.

Now, you can launch whichever version of Python you like within Anaconda Navigator by clicking on an environment name, then the green triangle next to that name (see above photo). You can choose to open the environment with base Python or with iPython, an interactive Python shell with improved functionality.

You could also choose to switch between environments using the source activate and source deactivate commands on the command line. You can view all available environments using the conda info --envs first, then copy the path to the one that you want to activate. Once you are done working that environment you can pass the deactivate command to return to your root environment.

Once you have the environment activated with your preferred version of Python, launching Python from the command line is as easy as typing python. You will see the >>> appear, indicating that you are in Python; when you are done, type quit() to return to bash. But can you honestly imagine writing and running entire analysis pipelines in this manner? Of course not. That’s why we can again turn to IDEs and text editors to make our lives easier.

Step Two: Selecting a text editor or IDE

IDEs are nicer to work with than text editors when coding in Python because they allow you to test chunks of code without running a whole script. Text editors like Atom and Sublime do have debugging functionality built-in, in that they will indicate when your code looks broken, but to figure out if you’ve actually written things the correct way you still need to run the whole shebang and see if it breaks (and if an error is printed, and if you even recognize the error). I’m still at the level of baby’s first Python script, so using an IDE is preferable as it lets me check things while I go (P.S. -this is a controversial opinion! There are those who argue in favor of learning to walk (command line + text editor) before you run (IDE). Choose whichever approach is best for your learning goals.).

Enter Anaconda, making life less complicated again. Anaconda comes bundled with a handful of IDE options. You can view them on the home tab:

If you like R Studio’s format you will probably like Spyder, which is visiually similar as it has windows for the console, script editor, and environment variables/help. This is what I started using when I was making the first tentative steps over from R to Python. My fellow Neurohackers recently introduced me to VScode, which I will be using going forward. Why? VScode is a one-stop-shop IDE that allows you to code in every language covered in this blog post, along with many others, as well. In the following VScode window I have open a bash shell, a R script, and a Python script that I am working on all at once, in the same program.

Keep in mind that the goal is to make coding easier for you, so you should feel free to pick an IDE that is the best fit. In the lab that I am in we have scripts written in all three languages so being able to swap seamlessly among them is ideal. If you are learning Python after having worked with R, you might prefer Spyder. And I’ve heard good things about another popular IDE (that does not come with Anaconda) called Pycharm. There’s enough options out there for you to be a veritable Python Goldilocks!

Step Three: Installing and managing packages and modules

Python packages are collections of functions, variables, and other assorted code that can be imported into your working enviornment to add different functionality. Modules are chunks of specific functions within a package. For example, if I wrote a package called ‘catplot’ each of the steps of interpreting imported data, plotting it, and then turning the individual data points into pictures of my cat’s face would be the modules in the package.

A neat thing about Python is that you can choose to import whole packages or individual modules into your working environment. Let’s practice this using Anaconda Navigator. Anaconda installs for you a variety of popular Python packages, which you can view underneath the ‘Environments’ tab. In your environment you should be able to find numpy and pandas in the list of installed packages. Now, open a new Python window and type in help(numpy) to view the documentation of this package.

…It didn’t work, did it? Because we didn’t import it yet. Like R, using packages and modules is a two-step process. Since Anaconda already installed these packages for us, we just need to import them (and like R, we need to import the packages we want to use every time we boot up a new Python window or script). import numpy. Now help(numpy) works and displays all of the package information (press q to quit out of the help documentation). Let’s try importing a module from a package next. To do this, tell Python first from which package the module should be loaded, then the name of the package itself. from pandas import melt. Now try help(melt) and help(pandas). The former works while the latter does not, because we only imported the module melt from pandas and not the whole package.

A brief aside

What if you want to work with packages that didn’t come installed with Anaconda? Open up a Python window and pass conda install $packagename. Note that this will install the package to your current Python environment; to install it to another, use the -n (environment name) flag: conda install -n $environment_name $packagename. A list of arguments for the conda install function are available here. If you are not using Anaconda, use the pip install $packagename command.

Another neat thing about Python is that you can set a nickname for the package that you are importing in order to make it easier to call functions from that package. This is why, if you’ve seen Python scripts before, you’ve probably seen import numpy as np at the top of the script. When you call modules from packages in Python, you type $package_name.$module_name. For example, the module melt from pandas that we just practiced importing is a function that reshapes data by gathering columns into rows (changing data from wide form to long form, essentially). An example of using that command in Python would look like longdata = pandas.melt(widedata). When writing a script, typing that many characters over and over can become tedious and is more prone to encouraging typos, so instead we can tell Python to import pandas as pd and make our commands somewhat shorter but more easier to type, longdata=pd.melt(widedata).

Finally, like languages and software, packages too have versions that may cause unexpected errors or conflicts. Thankfully, you can also manage which versions of packages you have installed in Anaconda Navigator. Across from the package name you can see which version is active, and whether there is an update available for that package (indicated by blue font and a blue arrow).

Clicking on the arrow gives you the option to update the packages individually in Anaconda. If you would like to go the other way, and roll back to a previous package version, click on the green checkmark next to a package name and select ‘mark for a specific version installation’ from the dropdown menu; a list of package versions will then be displayed for you to choose among. And, if you want to update all of your outdated packages in one fell swoop, pass conda update --all from the command line. Easy peasy.

Next Steps: Learn to code in Python

If you have installed Anaconda, you now have installed Python, a suite of poplar Python packages, and a couple of IDE options to work within. Now the fun part begins- learning to code and writing scripts! Luckily there are many, many fantastic resources for helping you learn this widely-used language. And never forget that when it comes to coding, especially in Python- the internet has your back.

TL;DR

Python is the language of data science, connecting scientists across a diverse range of fields and specialties. There is a bit of an uphill climb to writing in this language. Anaconda helps keep everything you need for working in Python under one GUI roof, so that getting started with coding is as simple as installing one program. Once you are ready to begin coding in Python you can check out the Neurohackademy intro to Python crash course or the online Python Data Science Handbook written by Neurohackademy instructor Jake VanderPlas.
